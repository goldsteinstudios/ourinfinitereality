How to Read Old Documents

If you encounter an earlier version:

  1. Check the version number. Pre-v0.986 documents may use "empty center" language.
  2. Note the date. Earlier documents may lack locked corrections.
  3. Prefer current version. Later versions correct earlier errors.
  4. Core claims are stable. V0 prohibition, O1 existence, and three requirements haven't changed - only their articulation has improved.

The Correction Principle

"Version history is evidence of honesty. A framework that never corrects itself either got everything right the first time (unlikely) or isn't paying attention to its mistakes."

RSM improves through explicit correction. Each version notes what changed and why. This is deliberate: we want mistakes to be visible and their fixes traceable.
